# Version v0.2.x

OVDS v0.2.x begins the next schema line with a generalized approvals model for regulatory and certification records.

This version keeps the existing role-based vehicle, powertrain, performance, and safety structure from v0.1.x, while extending `compliance` with a jurisdiction-agnostic approvals block.

The new approvals model is intended to represent records such as:

- EU type approval
- EPA certification
- Mexico NOM certification
- other authority-issued approval or conformance records

## Release Notes v0.2.0

OVDS v0.2.0 is a new schema line that introduces a generalized approvals structure under `compliance`.

### v0.2.0 Highlights

- Adds `compliance.approvals` for portable regulatory approval and certification records.
- Adds a reusable approval structure with authority, jurisdiction, scheme, domain, standard, identifier, status, and date fields.
- Adds `sourceDocs` so each approval can cite one or more official supporting documents instead of relying on a single URL field.
- Keeps approval records separate from `performance` so published technical results remain distinct from authority-issued classifications.
- Avoids duplicating vehicle lineage fields such as `modelYear` inside the approval object.
- Adds v0.2.0 examples that demonstrate EU, EPA, and Mexico NOM approval patterns.
- Expands `configuration.emissionsControls` with optional catalyst-system typing and placement details.
- Expands `configuration.emissionsControls` with optional emissions-sensor typing and placement details.
- Keeps the simpler emissions-controls fields for compatibility, while allowing richer published aftertreatment descriptions when sources provide them.

## Expected Evolution

Future v0.2.x releases may refine approval-domain vocabularies, add more jurisdiction-specific examples, and introduce additional reusable technical entities without changing the core purpose of `compliance.approvals`.
